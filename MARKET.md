# 시장 분석 — 국방 HILS 플랜트 모델 사업 전략

> 근거: deep-research 워크플로 2회 실행에서 3표 적대적 검증으로 확정된 주장(3-0 / 2-0)
> 기반. 종합 단계는 세션 리밋으로 스킵되어, 확정 사실을 직접 정리함. 각 사실에 출처
> 표기. **미확정·반증된 전제는 §5에 분리 기재** (정직성 원칙).

## 1. 핵심 발견 — 시장은 실재하나 전제 하나가 틀렸다

연구 착수 시 세운 전제 "리얼타임웨이브 = NI VeriStand 기반 통합업체"는 **반증되었다(3-0).**
리얼타임웨이브는 NI PXIe/VeriStand 재판매·통합이 아니라 **자체 실시간 OS 기반 플랫폼
RTNgine(2009년 출시)** 을 보유·사용한다 [realtimewave.com, realtimewave.co.kr].

→ **전략적 함의**: 국내 최강 경쟁자는 NI 생태계 밖에 있다. 즉 우리의 "NI VeriStand
`.so` 납품" 경로는 리얼타임웨이브와 정면충돌하지 않는 **차별화된 채널**일 수 있으나,
동시에 국내 방산 HILS의 상당수가 애초에 VeriStand가 아닌 RTNgine 위에서 돌아간다는
뜻이므로 — **모델을 특정 런타임에 묶지 않는 전략(FMU + C 코어)이 필수**다.

## 2. 확정된 시장 수요 근거

### 2-1. UAV급 6-DOF 플랜트 모델 수요는 실재한다 (3-0)
- KAIST–한화시스템 공동팀이 고정익 UAV HILS 환경을 구축: **Simulink 6-DOF 모델
  (랜딩기어 포함) + FLCC + RTNgine 미들웨어**. 공력계수는 **Datcom·AVL**로 추출.
  [KCI ART002901395]
- KAI(한국항공우주산업) 엔지니어가 2025년 논문 공저: **캐니스터 발사형 고정익 UAV**의
  비선형 시뮬레이션(플랜트) 모델이 필수이며 **HILS로 유도·제어 법칙을 검증**했다고 명시.
  [KCI ART003259423]

→ 우리 자산(6-DOF + 액추에이터 + 센서 + Dryden + 검증 하네스)이 정확히 이 수요
  형상과 일치. **특히 "Datcom/AVL 공력DB" 방식은 우리가 계수 테이블만 갈아끼우면
  고객 기체로 재타깃 가능** — 프레임 상품화의 핵심.

### 2-2. ADD·연구기관이 통합검증 인프라의 발주·수행 주체 (2-0)
- ADD 항공기술연구원 제1비행체계단이 무인기 체계 SIL을 구축·운용: **비행조종 HILS +
  항공전자 SIL + 데이터링크 + 임무장비 + GCS + FDAS**. [jkimst.org 2204]

### 2-3. 무인기 HILS는 국내에서 이미 반복 수행 중 (3-0)
- 리얼타임웨이브 실적: **2020 UCCR(무인전투기) FLCC HILS**, **2022 무인기 체계 표준
  프로토콜 통합 검증환경**. [realtimewave.co.kr/company-overview]

## 3. 경쟁 구도 — 인프라는 선점, 모델은 공백

- **인프라·리그 계층은 리얼타임웨이브가 강력히 선점**: KF-21 TFC SIL(2021), 전기리그·
  비행조종 ETS 실시간 시뮬레이션 컴퓨터(2019), Radar Interface OFP 환경(2018).
  유인기 HILS 리그 시장 진입은 사실상 레드오션. [realtimewave.co.kr/company-overview]
- **그러나 재사용 가능한 "플랜트 모델 카탈로그"는 어느 경쟁자도 상품으로 내세우지 않음
  (3-0)**: 리얼타임웨이브 사이트는 RTNgine·ICDSS·MissileNgine 등 **시험 인프라·ICD
  시뮬레이션**은 광고하나 6-DOF FDM/엔진/시커/센서 **플랜트 모델을 제품으로 나열하지
  않는다**. [realtimewave.com]

→ **핵심 포지셔닝**: "리그를 파는 시장"이 아니라 **"검증된 재사용 플랜트 모델을 파는
  시장"** — 여기가 비어 있다. 리그 통합업체(리얼타임웨이브 포함)조차 프로젝트마다
  모델을 새로 만들거나 고객이 늦게 주는 모델을 기다린다. 우리는 그들의 **경쟁자가
  아니라 모델 공급자(협력)** 로도 설 수 있다.

## 4. 납품 형태 표준 — FMU가 정답 (3-0, 2024+ 근거)

- NI VeriStand 공식 지원 플랜트 모델 형식 = **Simulink 생성 모델 + FMI 표준 준수 3rd
  party 모델**. 별도 "`.so` 표준"은 프레젠테이션에 명시되지 않음. [NI Streamlining 2024]
- VeriStand 실시간 모델링 환경 지원: Simulink·LabVIEW·**C/C++·FMI 2.0/3.0 co-sim**.
  [NI 2024]
- **VeriStand 2019+ 는 FMI 2.0 co-sim FMU를 Windows + PXI Linux RT 타깃에서 실행**
  (PXIe HILS 리그의 1급 납품 형식). [NI KB kA03q000000YGP0]
- FMI 3.0 지원 2024 Q1 제공. [NI 2024, 2-0]

→ **우리 M23~M24의 방향 재조정**: `.so`(VeriStand Model Framework)는 여전히 유효한
  하위 경로지만, **FMU 2.0/3.0 export가 1순위 납품 표준이어야 한다(M25로 승격).**
  FMU 하나면 VeriStand + RTNgine 외 툴(SCADE, Simulink, dSPACE)까지 커버 →
  §1의 "런타임에 묶지 않기" 전략과 정확히 부합.

## 5. 미확정·반증 항목 (정직성)

- ✗ **반증**: "리얼타임웨이브 = NI VeriStand 기반 통합업체" → 자체 RTNgine 사용 (3-0).
- ⏳ **미검증**(세션 리밋으로 표결 미완): LIG넥스원 군집무인기(에어소드) 사업 세부,
  ADD 모델 납품 형태가 Simulink 자동생성 C-코드인지 여부, VeriStand FMI가 co-sim
  전용인지 model-exchange 미지원인지. → 영업 진행 전 재확인 필요.
- 반증되지 않았으나 **단일 출처**인 항목(ADD SIL 구성 2-0)은 추가 교차확인 권장.

## 6. 결론 — 전략 및 우선 타겟

### 전략 타당성: **조건부 타당 (High, 단 납품형식 축을 FMU로 전환)**
소형 UAV 통합 플랜트 모델(6-DOF+센서+환경+**검증 하네스**)을 첫 상품으로 하는 전략은
수요(§2)·공백(§3)이 확정 사실로 뒷받침된다. **차별점은 모델이 아니라 "골든 검증된
모델 + 자동 인터페이스 명세 + CI 트레이서빌리티"** 패키지다 — 경쟁자가 상품화하지
않은 바로 그 계층. 단, §4에 따라 납품 1순위를 `.so` → **FMU**로 전환한다.

### 우선 타겟 고객 3곳
1. **HILS 리그 통합업체 (리얼타임웨이브 등) — "협력 공급" 채널.**
   그들은 리그를 갖췄고 모델 공백이 있다(§3). 경쟁이 아니라 **OEM 모델 공급자**로
   진입. RTNgine이 Simulink/FMU를 물릴 수 있으므로 FMU 납품이 열쇠. 진입장벽 최저.
2. **UAV 체계개발 주관 (KAI·한화시스템 계열) — "레퍼런스 플랜트" 채널.**
   §2-1에서 두 곳 모두 UAV 6-DOF+HILS를 직접 수행 중임이 확정. 자체 모델이 늦거나
   리그 선검증용 레퍼런스 플랜트가 필요한 구간을 공략. Datcom/AVL DB 교체형 프레임.
3. **ADD·연구기관 — "제3자 검증 모델" 채널.**
   §2-2에서 통합검증 인프라 발주·수행 주체로 확정. 체계업체 모델을 **검증할** 독립
   레퍼런스 모델 + 트레이서빌리티 문서 수요. 우리 골든 CI가 정확히 이 근거를 생산.
