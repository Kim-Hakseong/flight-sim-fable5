# QGroundControl 연동 테스트 & 사용 가이드 (초보자용)

> 이 문서 하나만 따라 하면, **다른 윈도우 PC**에서 이 프로젝트를 받아 QGroundControl로
> 시뮬레이터 비행체를 조종해볼 수 있습니다. 드론/ArduPilot을 처음 접하는 분을 기준으로
> 아주 쉽게 설명합니다.

---

## 0. 이게 뭐예요? (30초 설명)

- **비행 시뮬레이터**입니다. 진짜 드론/비행기 대신, 컴퓨터 안에서 도는 **가짜(시뮬) 비행체**예요.
- 이 가짜 비행체를 **QGroundControl**(줄여서 QGC)이라는 무료 지상관제 프로그램으로 조종합니다.
- 실제 드론을 QGC로 조종하는 것과 **똑같은 방식**(MAVLink 통신)으로 동작합니다. 그래서
  하드웨어 없이도 "QGC로 비행체를 띄우고, 미션을 넣고, 복귀시키는" 연습을 할 수 있어요.

그림으로 보면:

```
 [웹 브라우저: 3D 시뮬 화면]  ←→  [Bridge (Node 프로그램)]  ←→  [QGroundControl]
        비행체가 날아다님            통역사 역할              지도·버튼으로 조종
                                  (MAVLink 통신)
```

- **Bridge(브리지)** = 시뮬과 QGC 사이의 통역사. 시뮬의 상태를 QGC가 아는 언어(MAVLink)로
  바꿔주고, QGC의 명령을 시뮬로 전달합니다.

---

## 1. 준비물 설치 (윈도우)

세 가지만 설치하면 됩니다. 전부 무료입니다.

### ① Node.js (브리지를 돌리는 데 필요)
1. <https://nodejs.org> 접속 → **LTS** 버전 다운로드 → 설치(계속 "다음"만 누르면 됩니다).
2. 설치 확인: **명령 프롬프트(cmd)** 또는 **PowerShell**을 열고 아래 입력:
   ```
   node -v
   ```
   `v20.x.x` 같은 숫자가 나오면 성공.

> 이 프로젝트는 **추가 라이브러리 설치가 필요 없습니다**(`npm install` 안 해도 됨).
> Node.js 기본 기능만 씁니다.

### ② QGroundControl (관제 프로그램)
1. <https://qgroundcontrol.com> → **Download** → Windows용 설치.
2. 실행해서 지도 화면이 뜨면 준비 끝. (설정은 나중에 자동으로 됩니다.)

### ③ 코드 받기 (Git 또는 ZIP)
**방법 A — Git 사용 (권장):**
1. <https://git-scm.com/download/win> 에서 Git 설치.
2. 원하는 폴더에서 명령 프롬프트를 열고:
   ```
   git clone https://github.com/Kim-Hakseong/flight-sim-fable5.git
   cd flight-sim-fable5
   ```

**방법 B — Git 없이 ZIP:**
1. GitHub 저장소 페이지 → 초록색 **Code** 버튼 → **Download ZIP**.
2. 압축을 풀고, 그 폴더에서 명령 프롬프트를 엽니다.

## 2. 실행하기 (3단계)

### 1) 브리지 켜기
코드 폴더(`flight-sim-fable5`) 안에서:
```
npm run bridge
```
아래처럼 나오면 성공입니다. **이 창은 그대로 켜두세요.**
```
bridge up http=8765 gcs=127.0.0.1:14550
```
> 🔸 윈도우 **방화벽 팝업**이 뜨면 **"액세스 허용"**을 눌러주세요(안 그러면 QGC가 연결을 못 합니다).

### 2) 시뮬 화면 열기
웹 브라우저(Chrome 권장)를 열고 주소창에:
```
http://localhost:8765
```
3D 비행체가 활주로에 있는 화면이 나오면 성공.

### 3) QGroundControl 켜기
QGC를 실행만 하면 **자동으로 연결**됩니다(별도 설정 불필요).
잠시 후 지도에 비행기 아이콘이 나타나고, 상단에 **비행 모드 이름**(예: Manual)이 보이면 연결 완료!

> 연결이 안 되면 → 이 문서 맨 아래 **"문제 해결"** 참고.

## 3. QGC 화면 둘러보기

- **상단 바**: 현재 모드(Manual/Auto/RTL 등), 배터리, GPS 위성 수.
- **왼쪽 위 버튼들**: 이륙(Takeoff), 복귀(RTL/Return), 일시정지(Pause).
- **지도**: 비행체 위치와 이동 궤적(빨간 선).
- **아래 바**: 고도, 속도, 방향 등 텔레메트리 숫자.
- **왼쪽 위 Q 아이콘 → Plan**: 미션(웨이포인트)과 **Fence(비행금지구역)**를 그리는 화면.

## 4. 따라 하기 (기본 조작 실습)

아래 순서대로 하나씩 해보세요. 각 동작이 무슨 뜻인지도 같이 적었습니다.

### ① ARM (시동 걸기)
- **ARM = 모터 시동**. 안전을 위해 시동을 안 걸면 안 움직입니다.
- QGC 상단/하단의 **Arm** 버튼을 누르거나, 슬라이드로 확인.
- 성공하면 상단이 "Armed"로 바뀌고, 시뮬 화면 좌측 HUD에 `ARMED`가 뜹니다.

### ② 이륙 (Takeoff)
- 왼쪽 위 **이륙(Takeoff)** 버튼 → 목표 고도 확인 → 슬라이드.
- 비행체가 활주로를 달려 **이륙**하고 지정 고도로 올라가 선회합니다.

### ③ 모드(Mode) 바꿔보기
모드 = 비행체가 **누구 말을 듣는지**입니다. 상단 모드 메뉴에서 선택:
- **Manual**: 사람이 직접(여기선 키보드로) 조종.
- **Auto**: 업로드한 **미션(웨이포인트)** 을 스스로 따라 비행.
- **Guided**: "여기로 가"라고 지도에서 찍은 한 점으로 비행.
- **RTL** (Return To Launch): **출발지로 자동 복귀** 후 착륙.
- **Loiter**: 현재 위치에서 **제자리 선회**(고정익이라 원을 그리며 돎).

### ④ "여기로 이동" (Guided go-to)
- 지도에서 아무 곳이나 **길게 누르거나 우클릭** → **"Go to location"**.
- 비행체가 그 지점으로 날아갑니다.

### ⑤ 미션 넣기 (웨이포인트 비행)
1. 왼쪽 위 **Q → Plan** 화면으로 이동.
2. 지도를 클릭해서 **경유지(웨이포인트)** 를 여러 개 찍습니다.
3. 오른쪽 위 **Upload**(업로드)로 비행체에 전송.
4. 다시 비행 화면으로 와서 모드를 **Auto**로 바꾸면 경로를 따라 비행합니다.

> ❓ **"미션 경로가 빙글빙글 크게 도는데 고장인가요?"** → **정상입니다.**
> 고정익(비행기)은 헬기처럼 제자리에서 못 돕니다. 최소 선회 반경이 있어서, 경유지가
> 촘촘하거나 급하게 꺾이면 크게 원을 그리며 돌아 다시 접근합니다.

### ⑥ 복귀 (RTL)
- **복귀(Return)** 버튼을 누르면 출발지로 돌아와 자동 착륙합니다.

### ⑦ 비행금지구역 (Geofence) — 이 프로젝트의 기능
1. **Q → Plan → Fence** 탭 선택.
2. **원형** 또는 **다각형** 금지/허용 구역을 그리고 **Upload**.
3. 비행 중 비행체가 그 경계를 넘으면 → **자동으로 복귀(RTL)** 하고, QGC에 **경고 메시지**가 뜹니다.
   - **Inclusion(포함)**: 이 안에만 있어야 함(밖으로 나가면 위반).
   - **Exclusion(제외)**: 이 안에 들어가면 안 됨.
   - **고도 상한**: 설정 고도(FENCE_ALT_MAX)를 넘으면 위반.

### ⑧ 파라미터 바꿔보기 (고급)
- QGC의 **차량 설정(Vehicle Setup) → Parameters** 에서 값 검색/수정 가능.
- 예: `AP_` 로 시작하는 값들은 오토파일럿 게인입니다. 바꾸면 비행 특성이 즉시 달라집니다.

### ⑨ 고장 주입 (HILS 체험) — 시뮬 화면에서
시뮬 화면 아래쪽 버튼으로 **센서/서보 고장**을 넣어볼 수 있습니다
(GPS DROP, GYRO BIAS, PITOT DROP, AIL JAM 등). 넣으면 QGC에도 **센서 경고**가 뜨고,
비행체가 그 고장에 어떻게 반응하는지 볼 수 있습니다. `CLEAR ALL`로 해제.

## 5. 시뮬 화면 키보드 단축키

브라우저의 3D 시뮬 화면을 클릭한 뒤 사용:

| 키 | 기능 |
|---|---|
| `Space` | ARM / DISARM (시동) 토글 |
| `T` | 자동 이륙 |
| `M` | Manual 모드로 전환 |
| `↑` `↓` | 피치(기수 상하) |
| `←` `→` | 롤(좌우 기울기) |
| `A` `D` | 요(좌우 방향) |
| `W` `S` | 스로틀(엔진 출력) 증가/감소 |
| `C` | 채널 모니터(신호값) 켜기/끄기 |
| `E` | 엔지니어링 콘솔(상태·추정오차·차트) |
| `R` | 리셋(활주로에서 다시 시작) |

## 6. 초보자 용어 사전

- **MAVLink**: 드론과 관제 프로그램이 주고받는 **공통 통신 언어(프로토콜)**.
- **ArduPilot**: 대표적인 **오픈소스 오토파일럿(자동조종 소프트웨어)**. 이 시뮬은 QGC가
  ArduPilot 비행기(ArduPlane)로 인식하도록 흉내 냅니다.
- **GCS** (Ground Control Station): **지상관제 프로그램**. 여기선 QGroundControl.
- **텔레메트리(Telemetry)**: 비행체 → 관제로 올라오는 **상태 정보**(위치·자세·배터리 등).
- **ARM / DISARM**: 모터 **시동 걸기 / 끄기**.
- **모드(Mode)**: 비행체의 **행동 방식**(Manual/Auto/Guided/RTL/Loiter…).
- **웨이포인트(Waypoint)**: 미션의 **경유 지점**.
- **RTL**: **출발지 자동 복귀**.
- **지오펜스(Geofence)**: **비행 허용/금지 구역**. 넘으면 자동 회피(복귀).
- **HILS** (Hardware-in-the-Loop Simulation): 실제 장비/제어기를 **시뮬 비행체와 연결해**
  시험하는 방법. 이 프로젝트가 지향하는 용도입니다.

## 7. 문제 해결 (FAQ)

**Q. QGC에 비행체가 안 떠요 / 연결이 안 돼요.**
1. 브리지 창에 `bridge up ...` 이 떠 있나요? 안 떠 있으면 `npm run bridge` 다시 실행.
2. 윈도우 **방화벽에서 Node.js를 허용**했나요? (처음 실행 때 팝업 → 허용)
3. QGC를 **껐다 켜** 보세요. QGC는 켜질 때 UDP 14550으로 자동 연결을 시도합니다.
4. 백신/보안 프로그램이 UDP를 막고 있진 않은지 확인.

**Q. `npm run bridge` 에서 "node를 찾을 수 없습니다" 오류.**
→ Node.js가 설치 안 됐거나, 설치 후 **명령 프롬프트를 새로 열지** 않은 경우입니다. 창을 닫고 다시 여세요.

**Q. `http://localhost:8765` 페이지가 안 열려요.**
→ 브리지가 꺼져 있거나 다른 프로그램이 8765 포트를 쓰는 중입니다. 브리지 창을 확인하세요.

**Q. 포트를 바꾸고 싶어요 / QGC가 다른 PC에 있어요.**
→ 브리지 실행 시 환경변수로 지정할 수 있습니다(윈도우 PowerShell 예):
```
$env:BRIDGE_HTTP_PORT=9000; $env:GCS_HOST="192.168.0.50"; npm run bridge
```
- `GCS_HOST` = QGC가 돌아가는 PC의 IP (같은 PC면 안 바꿔도 됨).
- `GCS_PORT` = 기본 14550.

**Q. 복귀(RTL)나 미션 중에 착륙하면서 살짝 기울어요.**
→ 정상입니다. 바람/난류 속 착륙이라 약간 기웁니다. 완전히 뒤집힐 정도만 아니면 정상 동작입니다.

## 8. 개발자용 메모 (구조 & 디버깅)

- **포트**: HTTP `8765`(시뮬 페이지 + 텔레메트리 수신), MAVLink UDP `14550`(QGC).
- **핵심 파일**:
  - `bridge/server.mjs` — HTTP/SSE ↔ MAVLink UDP 브리지.
  - `bridge/mavlink.mjs` — MAVLink v1/v2 인코딩·디코딩.
  - `src/vehicle.js` — 시뮬 비행체(물리+오토파일럿+추정기+지오펜스)의 단일 구현.
  - `src/geofence.js` — 지오펜스 기하 판정(순수 함수).
- **브리지 로그 읽기**: 브리지 창에 `gcs → bridge: ...`(QGC가 보낸 명령), `bridge → sim: ...`
  (시뮬로 전달한 명령)이 실시간으로 찍힙니다. **어떤 버튼이 무슨 명령을 보냈는지** 여기서 보입니다.
- **테스트 돌려보기**(개발 시):
  ```
  npm test              # 단위 테스트
  npm run check:gcs     # 브리지 + 가짜 GCS로 MAVLink 왕복 검증
  npm run hils          # HILS 시나리오 벤치
  ```
- 자세한 아키텍처/규범은 저장소 루트의 `PRD.md`, `CLAUDE.md` 참고.
